FRONTEND JAVASCRIPT (static/script.js)
================================================================================

Navigation Functions:
---------------------
homeBtn event listener
  Navigates the user to the home page (/) when the Home button is clicked.

reqcrtBtn event listener
  Navigates the user to the request certificate page when the Request 
  Certificate button is clicked.

contactBtn event listener
  Navigates the user to the contact page when the Contact Us button is clicked.

API Communication Functions:
----------------------------
backendBtn event listener
  Fetches a greeting message from the /api/hello endpoint and displays it in 
  the response box on the page.

dataForm event listener
  Captures user input from the form and sends it to the /api/data endpoint 
  as JSON, then displays the server's confirmation response.

displayResponse(message, elementId)
  Displays a message in a specific HTML element by updating its text content 
  and making it visible with CSS styling. Used to show API responses or errors 
  to the user.


BACKEND PYTHON (app.py)
================================================================================

Page Routes:
------------
index()
  Returns the main home page template (index.html) when a user visits the 
  root URL (/). Serves the entry point of the application.

request_certificate()
  Returns the certificate request form page template (reqcrt.html) when a user 
  navigates to /request-certificate. Allows users to submit certificate requests.

contact()
  Returns the contact form page template (contact.html) when a user navigates 
  to /contact. Allows users to send inquiries or feedback.

API Endpoints:
--------------
hello()
  GET endpoint (/api/hello) that returns a simple JSON greeting message from 
  the backend to test basic API communication. Returns: {'message': 'Hello from 
  Python backend!'}

receive_data()
  POST endpoint (/api/data) that accepts JSON data from the frontend and 
  returns a confirmation response. Returns: {'status': 'success', 'received': 
  [data]}


CSS STYLESHEET (static/style.css)
================================================================================

Layout & Structure Classes:
---------------------------
.header-container
  Main navigation header with gradient background and rounded corners. Creates 
  the top banner containing the app title and navigation buttons.

.button-container
  Flexbox container for navigation buttons using transparent background to 
  seamlessly blend with the header. Distributes buttons evenly across the header.

.container
  White content wrapper for page sections with rounded corners and shadow. 
  Provides visual separation of content from the gradient background.

Text & Spacing Classes:
-----------------------
body
  Global page styling with blue-cyan gradient background, sets font family and 
  centers all content using flexbox.

header (inside .container)
  Purple gradient background for page headers with white text and padding. 
  Distinguishes content sections visually.

main
  Content area padding to create proper spacing within the container.

section
  Wrapper for form sections with bottom margin for vertical separation between 
  different content blocks.

Form & Input Classes:
---------------------
#dataForm
  Layout container for form inputs and submit button displayed in a row using 
  flexbox.

input[type="text"]
  Text field styling with border and focus effect. Expands to fill available 
  space and highlights in blue when focused.

.response-box
  Display container for API responses and messages. Hidden by default and shown 
  via JavaScript when data is available.

.response-box.show
  CSS class toggled by JavaScript to display response messages to the user with 
  visible styling.

Button Styling Classes:
-----------------------
button
  Default button styling with transparent background and white text. Includes 
  hover animation that lifts buttons slightly.

#backendBtn
  Special purple gradient styling for the backend test button to distinguish 
  it from navigation buttons.

button:hover
  Hover effect that lifts button up and adds shadow for interactive feedback.

button:active
  Active/click state that returns button to original position for tactile 
  feedback.


HTML TEMPLATES
================================================================================

Page Structure (All pages):
---------------------------
index.html
  Main home page containing API testing section with backend button and form 
  section to send user data to the backend.

reqcrt.html
  Certificate request page with form for collecting user name and email address

contact.html

Meta Tags:
----------

  Declares UTF-8 character encoding for proper text rendering across all pages.

<meta name="viewport">
  Ensures responsive design by setting viewport width to device width and 
  initial zoom level to 1.0.


EXTERNAL LIBRARIES & FRAMEWORKS
================================================================================

Flask (app = Flask(__name__))
  The main web framework used to create routes, handle HTTP requests, and 
  serve templates. Enables the creation of API endpoints and page serving.

CORS (CORS(app))
  Allows Cross-Origin Resource Sharing so the frontend can make requests to 
  the backend from different domains. Essential for local development and 
  future deployment scenarios.

render_template(template_name)
  Flask function that loads and renders HTML template files from the 
  templates/ directory and returns them to the user's browser.

jsonify(data)
  Flask function that converts Python dictionaries into JSON format for API 
  responses. Automatically sets the correct Content-Type header.

load_dotenv()
  Loads environment variables from the .env file so sensitive configuration 
  (like debug mode) can be managed securely without hardcoding values.

fetch(url, options)
  JavaScript Fetch API that makes HTTP requests to backend endpoints. Used 
  for GET requests (reading data) and POST requests (sending data).


APPLICATION FLOW
================================================================================

User Interaction → Frontend Handler → API Request → Backend Processing → 
Response → Display to User

Example: User clicks "Message from backend" button
  1. backendBtn event listener is triggered
  2. fetch() sends GET request to /api/hello
  3. Backend hello() function processes request
  4. Backend returns JSON response
  5. displayResponse() shows message to user

Example: User navigates using buttons
  1. homeBtn/reqcrtBtn/contactBtn is clicked
  2. Event listener triggers window.location.href to route
  3. Flask backend renders appropriate HTML template
  4. New page loads with navigation maintained

Example: User submits a form
  1. Form submit event listener triggers
  2. Form data is collected and serialized to JSON
  3. fetch() sends POST request to backend API endpoint
  4. Backend processes data and returns confirmation
  5. Response is displayed in .response-box container
